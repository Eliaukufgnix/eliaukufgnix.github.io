# 算法

# 贪心法

  • 贪心法是一种选择当前最优解的策略，希望通过每一步的局部最优选择，达到全局最优解。
  • 在贪心算法中，每一步都会考虑当前情况下的最佳选择，而不考虑未来可能的影响。
  • 贪心算法通常适用于满足"贪心选择性质"和"最优子结构"两个性质的问题。
  • 优点是算法简单、高效，并且常常能够快速得到较优解。然而，由于贪心法的局限性，它并不能保证获得全局最优解。
  • 贪心法通常用于优化问题，例如最小生成树、背包问题

# 最小生成树

最小生成树是在加权连通图中找到一棵包含所有顶点的树，使得树上所有边的权重之和最小。

# 背包问题

背包问题是一个组合优化问题，要求在给定容量的背包中，选择一些物品放入背包中，使得物品的总价值最大化，同时不能超过背包的容量限制。

# 分治法

  • 分治法是将一个大问题拆分成若干个相同或相似的小问题，然后递归地解决这些小问题，最后将结果合并得到原问题的解。
  • 分治法通常包括三个步骤：分解（Divide），解决（Conquer），和合并（Merge）。
  • 在分治算法中，原问题被分解为多个子问题，各个子问题独立求解，然后将子问题的解合并成原问题的解。
  • 优点是能够解决复杂的问题，尤其是分解成多个相同或相似的子问题时。但是，递归处理可能会导致较大的开销和额外的空间复杂度。
  • 分治法通常用于分割问题、排序问题等，例如归并排序、快速排序。

# 冒泡排序

public static int[] BSort(int[] arr){
    int n=arr.Length;
    for(int i=0;i<n-1;i++){
        for(int j=0;j<n-1-i;j++){
            if(arr[j]>arr[j+1]){
                int temp=arr[j];
                arr[j]=arr[j+1];
                arr[j+1]=temp;
            }
        }
    }
    return arr;
}

# 快速排序

public static class QSort
{
    public static void QuickSort(int[] arr,int left,int right)
    {
        if (left < right)
        {
            int pivot= Partition(arr, left, right);
            QuickSort(arr, left, pivot - 1);
            QuickSort(arr, pivot +1, right);
        }
    }
    public static int Partition(int[] arr, int low, int high)
    {
        int pivot = arr[high];
        int i = low - 1;
        for (int j = low; j < high; j++)
        {
            if (arr[j] <= pivot)
            {
                i++;
                Swap(arr, i, j);
            }
        }
        Swap(arr, i + 1, high);
        return i + 1;
    }
    static void Swap(int[] arr, int i, int j)
    {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}

public int Func(int n)
{
